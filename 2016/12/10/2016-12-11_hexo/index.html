如何用Hexo搭建一个博客

2016-12-10
Hexo

作为一个程序员或技术研究者，每每解决了一个问题或发现了一个新的技术点，自带闷骚属性的猿们往往都抑制不住心里的“骚”动，想找个人倾（炫）诉（耀），可惜的是程序员与生俱来的“找不到对象”光环，最终都是陷入无处倾诉的地步（哪怕有对象你若跟对方说代码的事情，难免被对方反手就是一个按太阳穴轮刮眼眶赏了个耳聪目明）；有苦不能说，有X不能装是最痛苦的！所以，you,need a place to zhuangbi!哎~那搭个博客
，写几篇文章，放上你的Github链接，一言不合还可以在尾部放个支付宝什么的，不就满足了你心里的那股骚动了吗？所有说，博客对于程序员来说，意义就在于~引用某乎的说法：与世界分享你的知识、经验和见解（还有逼格）。

前言

那么建个博客是很简单的，但是程序员往往爱折腾爱作死，想自己搭。博客，说白了不就个网页吗，搞个服务器，搞点html，发布上去，不就完了。但是！好死不死你跟我一样是一名安卓开发者！又好死不死你跟我一样不懂服务器还不懂前端，网上一搜，别人的博客是用WordPress搞的动态博客，说啥都要一个服务器，好死不死你又跟我一样穷，那么，梦想破灭了吗？那倒不至于，有动就有静嘛~如果你是一个不懂服务器又不懂前端而且又想做自己的博客，可以，请跟着我的脚步，我会带你“真真正正”做一个属于你自己的博客。

个人提醒

虽然这个没什么难度，但是这东西需要大量的时间去折腾,所以搞之前建议做好心理准备。

先说说Github

Github的优点

  • GitHub 是一个面向开源及私有软件项目的托管平台，用Git进行版本控制（比SVN高到不知哪里去了）。
  • Github 上面大牛云集，有事没事泡一泡，能学到很多知识。
  • 使用Github的Github Page来搭建博客，是免费的。

    Github Page

    Github Pages可以被认为是用户编写的、托管在github上的静态网页。

    使用Github Page的好处

  • 呃。。。如上所说，免费~免费~免费。
  • 可以绑定域名(但暂时只能绑定一个)。

安装步骤也简单，一直下一步到底就行，并且安装后也无需配置环境变量。

#第三步:Hexo
Hexo 是一个基于nodejs 的静态博客网站生成器，作者是来自台湾的 Tommy Chen。

特点：

  • 不可思议的快速 ─ 只要一眨眼静态文件即生成完成。
  • 支持 Markdown。
  • 仅需一道指令即可部署到 GitHub Pages 和 Heroku。
  • 已移植 Octopress 插件。
  • 高扩展性、自订性。
  • 兼容于 Windows, Mac & Linux。
